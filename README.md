
# NCCU 社團管理系統

[cite_start]本系統為針對大學生設計的社團管理平台，支援**社團會員管理、社產管理與財務管理**三大核心模組 [cite: 1, 2][cite_start]。使用者可登入後根據加入的社團及學期進行各項管理，並透過前後端串接與資料庫操作實現動態內容展示。

---

## 專案願景與動機
[cite_start]在校園社團營運中，傳統的 Excel 紀錄常導致社產遺失、財務帳目不清及資訊交接斷層 [cite: 1, 2][cite_start]。本專案旨在透過數位化流程優化，提供一個直觀且具備權限管控的平台，提升行政效率並確保社團資產的透明化管理。

---

    * 負責設計 **ER Diagram (ERD)** 與 **Relational Model**，建立嚴謹的關聯式資料庫結構，確保數據的一致性與擴充性。
* **核心功能實作 — 社產管理模組**：
    * 負責「社產管理」功能的後端邏輯與前端介面開發。
    * 實作**自動化借還登記系統**與**狀態追蹤功能**，大幅降低人工點收的時間成本，展現「將麻煩事變簡單」的實作精神。

## 功能說明

  ### 使用者功能
  - 使用學號與密碼登入系統
  - 根據學期查看已加入社團
  - 點擊社團卡片進入社團頁面
  - 點擊頭像可查看與編輯個人資料

  ### 社團管理功能（依角色授權）
  - **會員管理**
    - 會員總覽、會員個資編輯、新增成員、權限設定
  - **社產管理**
    - 社產列表、詳細資料、借還登記、新增/編輯社產
  - **財務管理**
    - 個人繳費紀錄、繳費總覽、單一會員繳費明細

## 啟動方式

  ### 安裝套件
  ```bash
  npm install
  ```

  ### 啟動伺服器
  ```bash
  node server.js
  ```
  伺服器預設執行於 `http://localhost:3001`，靜態頁面會由 `/public` 提供。

## 環境配置注意事項

為了確保系統正常運作，請在啟動前完成以下配置：

* **資料庫環境**：
    * 本系統使用 **MySQL** 作為資料存儲，啟動前請確保本地 MySQL 服務已開啟。
    * 執行根目錄下的 `dbms-project.sql` 腳本，以建立核心資料表結構（包含 `Member`, `Club`, `ClubMember` 等）。
    * **建議做法**：在 TablePlus 或常用資料庫管理工具中建立名為 `dbms-project` 的資料庫，並匯入上述 SQL 檔案。

* **連線參數設定**：
    * 請至 `backend/db.js` 修改資料庫連線資訊，包括 `user`、`password` 及 `database` 名稱，以符合您的本地環境。

* **測試帳號說明**：
    * **管理員測試帳號**：`110306001`。
    * 該帳號已預設配置多個社團之管理權限，建議以此帳號登入以完整體驗「會員、社產、財務」三大模組功能。

* **系統開發備註**：
    * 系統前端頁面之行為控制與 API 串接邏輯主要位於 `public/club.js`，開發邏輯與命名規範參考自 `club.html`。
